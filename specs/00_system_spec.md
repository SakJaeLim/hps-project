# 00. 시스템 명세 (System Spec)

## 목적
선박 적재계획 요청에 대해 제약을 충족하는 적재 권고안을 근거와 함께 자동 생성.
1차 목표는 **현재 기술의 통합·활용**(ADR-0001), 최적화 성능 경신이 아니다.

## 범위 (1개월 MVP)
- IN: 시뮬 데이터, 온톨로지 코어+제약 5종, **RL 적재 엔진(원우 모델 재사용)** + Greedy 기준선, 4-에이전트 루프, RAG+LoRA 1회, 대시보드 4화면, 평가
- OUT(차기): **CP-SAT 비교·교체(ADR-0001)**, 실시간 스트리밍, 본격 RL 재학습, Edge INT4, 멀티 선석

## 핵심 흐름
운영자 요청 → ①상태인식 → ②계획(엔진) → ③제약검증(Cypher) → ④설명(SLM) → 대시보드. 위반 시 ②로 재시도.

## 비기능 요구
- 권고 응답 ≤ 5초 / 제약 위반율 < 1% / 재취급: 기준선 대비 비교(참고·시뮬)
- 1차 성공 기준 = 유효 계획 생성·전 흐름 작동(통합). 재취급 ≥15%는 향후 연구 목표.

## 제약·의존성
- SLM LoRA 파인튜닝은 ~24GB GPU(Colab/Runpod) 필요 — 1주차 확보.
- RL 엔진은 도메인 전문가(원우) 학습 모델·프레임워크에 의존(체크포인트·env 사양 수령).
